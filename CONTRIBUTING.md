# Contributing to Trackly

Thanks for your interest in contributing! Trackly is a small, focused tool — contributions that keep it simple and coach-friendly are most welcome.

## Running locally

```bash
npm install
npm run dev
```

The app runs entirely in the browser with no backend. State is persisted in `localStorage`.

## Branch and PR flow

1. Fork the repo and create a branch from `main` (e.g. `fix/score-display` or `feat/export-csv`).
2. Make your changes and verify them manually in a browser (desktop and mobile if relevant).
3. Open a pull request against `main`. Fill in the PR template — a short description and how to test is enough.

There are no automated tests right now, so clear manual-test instructions in your PR are important.

## What contributions are welcome

- Bug fixes
- Mobile / accessibility improvements
- UX polish that keeps the interface simple for coaches
- Performance improvements
- Translations (the UI is currently in German)

Please open an issue before starting large features or structural changes so we can discuss the direction first.

## Opening issues

Use the issue templates on GitHub — they keep reports actionable. If you're unsure whether something is a bug or a feature, open an issue and describe what you observed.
